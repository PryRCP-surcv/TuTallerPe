Objetivo del avance

Elevar fidelidad y preparar crecimiento

El objetivo principal fue transformar el prototipo existente en una experiencia de alta fidelidad, accesible, consistente y apta para pruebas con usuarios, sin romper los flujos ya definidos. En paralelo, se preparo la base tecnica del backend para futuros avances con Spring Boot.

Metodologia

Como se trabajo el Avance 3

  • Revision del proyecto HTML, CSS y JavaScript existente.
  • Redisenio visual del frontend con enfoque responsive y accesible.
  • Validacion de formularios clave y mejora de estados de interfaz.
  • Preparacion de paginas de apoyo para pruebas, heuristicas y accesibilidad.
  • Definicion de una arquitectura inicial de backend desacoplada en Spring Boot.
Usuarios evaluados

5 perfiles considerados

Se contemplaron estudiantes, profesionales jovenes, madres de familia, docentes y organizadores culturales para validar distintos escenarios de uso.

Tareas

5 tareas centrales

Registrarse, buscar taller, ver detalle, inscribirse y publicar taller como administrador.

Resultados

Hallazgos mas relevantes

  • La navegacion mejoro al incorporar breadcrumbs y pagina activa en el menu.
  • Los usuarios completaron mas rapido la comparacion de talleres gracias a tarjetas con informacion resumida.
  • Las validaciones redujeron errores en login, registro y publicacion de talleres.
  • La pantalla de ayuda y la documentacion fortalecen la presentacion academica del proyecto.
Problemas encontrados

Aspectos detectados durante la revision

  • Inconsistencia visual entre pantallas del proyecto original.
  • Ausencia de estados claros en formularios y confirmaciones.
  • Falta de apoyo para teclado, breadcrumbs y ayudas contextuales.
  • Documentacion insuficiente para sustentar pruebas y evaluacion heuristica.
Evaluacion heuristica

Principales aportes

La aplicacion de las 10 heuristicas de Nielsen evidencio la necesidad de prevenir errores, reforzar consistencia, mostrar estado del sistema y brindar ayudas explicitas. Estas observaciones orientaron varias de las decisiones de diseno del nuevo prototipo.

Accesibilidad

Decisiones inclusivas incorporadas

Se mejoraron contraste, jerarquia de encabezados, foco visible, labels asociados, atributos aria, textos alternativos y soporte para navegacion por teclado.

Mejoras del prototipo

Cambios mas visibles

El prototipo ahora muestra una identidad visual mas solida, estados de botones y formularios, cards informativas, ayudas contextuales, flujo administrativo mas convincente y documentacion complementaria para la sustentacion.

Conclusiones

Cierre del avance

El Avance 3 deja un frontend listo para presentar y evaluar, asi como una base backend escalable y desacoplada. La siguiente etapa deberia conectar ambos frentes, persistir datos reales y robustecer la seguridad del sistema.
